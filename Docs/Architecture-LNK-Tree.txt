# Hippocrates 360 – LNK Tree Architecture

Този документ описва аспектското дърво (`TVirtualStringTreeAspect`) и бинарния формат на `.lnk` файла, върху който то е базирано.

Аспектското дърво е оптимизирано за:

- директна работа върху memory-mapped файл  
- минимални алокации в паметта  
- бързо обхождане и филтриране  
- използване на `TVtrVid` като „тип на възела“ за логика и UI

---

## 1. Общ преглед

LNK файлът представлява последователност от фиксирани възли с дължина **64 байта**.

Всеки възел се състои от:

- **node част** – 52 байта (`lenNode = 52`)  
- **data част** – 12 байта (засега), започваща на offset 52 (`LenData = 64`)

const
  LenData = 64;  // обща дължина на възела в .lnk файла
  lenNode = 52;  // дължина на самия VirtualTree възел
